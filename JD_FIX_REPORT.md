## 1. 포지션명 텍스트 위치가 너무 위에 있음

### 현황

```
TEMPLATE 구조 (868×4160px)
═══════════════════════════
y=0     ┌─────────────────┐
        │  QR + iyuno logo │
y=265   │  (로고 텍스트 끝) │
        ├─ ─ ─ ─ ─ ─ ─ ─ ┤
        │    원본 빈 공간    │  ← 126px 간격 (265→391)
        │                   │
y=391   │  채용공고 텍스트    │
        │  monster + banner │
y=700   └─────────────────┘
```

### 문제 분석

| 항목 | 템플릿 원본 | 현재 코드 기본값 |
|------|-----------|----------------|
| 로고 끝 → 채용공고 시작 간격 | **126px** (y=265 → y=391) | **55px** (`positionStyle.gap`) |
| 코드의 로고 슬라이싱 끝 | - | `TOP_LOGO_END = 275` |
| 코드의 배너 슬라이싱 시작 | - | `TOP_BANNER_START = 310` |

**원인**:
1. 코드의 `gap` 기본값이 **55px**인데, 템플릿 원본의 실제 간격은 **126px**. 공간 자체가 원본의 절반도 안 됨.
2. 텍스트 Y 오프셋 (`positionStyle.y = 38`)이 55px 공간 안에서 상단에 치우침.
3. `TOP_LOGO_END = 275`는 정확하지만, `TOP_BANNER_START = 310`이 실제 채용공고 텍스트 시작(y=391)보다 **81px 위**에서 슬라이싱. 즉 "채용공고" 위의 빈 공간이 코드에서 잘려나감.

### 해결 방안

**방안 A**: `gap` 기본값을 126px로 변경하고, `y` 오프셋 조정하여 텍스트를 간격 중앙에 배치
- 코드 변경: `positionStyle: { fontSize: 20, gap: 126, x: 60, y: 70 }`
- Y 오프셋 슬라이더 max도 120 이상으로 확대 필요

**방안 B**: 슬라이싱 좌표 재조정
- `TOP_BANNER_START`를 310 → 391로 변경 (빈 공간을 gap에 포함)
- gap 기본값을 자연스럽게 조정

**권장**: 방안 A가 단순. gap 기본값만 늘리면 됨.

## 2. 섹션 헤더 vs 전형절차 글자 크기 차이

### 현황

```
DYNAMIC SECTION (Canvas 텍스트)          BOTTOM SECTION (템플릿 이미지)
══════════════════════════════          ══════════════════════════════
채용분야 및 인원  ← HEADER_SIZE=24px    전형절차  ← 템플릿 이미지 내 ~30px
소개             ← 24px                HEALTH & LIFE STYLE
담당업무          ← 24px                WORK & DEVELOPMENT
자격요건          ← 24px
```

### 측정 결과

| 텍스트 | 위치 | 글리프 높이 (868px 기준) | 추정 폰트 크기 |
|--------|------|----------------------|--------------|
| 채용분야 및 인원 (템플릿 원본) | y=819-848 | **30px** | ~30px |
| 소개 (템플릿 원본) | y=986-1015 | **30px** | ~30px |
| 담당업무 (템플릿 원본) | y=1139-1168 | **30px** | ~30px |
| 전형절차 (템플릿 원본) | y=1939-1968 | **30px** | ~30px |
| Canvas에서 그리는 헤더 | 동적 | HEADER_SIZE=**24px** | 24px |

**결론**: 템플릿 이미지에 박힌 모든 섹션 헤더(전형절차 포함)는 **~30px** 크기인데, Canvas에서 동적으로 그리는 헤더는 **24px**로 **6px 차이**.

### 해결 방안

`HEADER_SIZE`를 24 → **30**으로 변경하면 템플릿의 전형절차와 크기 일치.
`HEADER_LH` (line height)도 32 → **38** 정도로 함께 조정 필요.

## 3. 테두리 윤곽선이 텍스트 박스에 의해 가려지는 이슈

### 현황

템플릿 이미지(700-1860 영역)에는 섹션 구분선과 테두리가 포함되어 있음:

```
TEMPLATE 700-1860 영역 (사용되지 않음!)
══════════════════════════════════════
y=769-770  ─── 구분선 (가로) ───
y=819-848  채용분야 및 인원  (텍스트)
           ·····(빈 공간)·····
y=986-1015 소개
           ·····
y=1139-1168 담당업무
           ·····
(... 계속 ...)
```

**하지만** 현재 코드는 이 영역(700-1860)을 **완전히 건너뛰고**, 대신:
1. 흰 배경을 깔고
2. 그 위에 Canvas로 텍스트를 직접 그림
3. 맨 위에 구분선 1줄만 그림 (코드 296-302줄)

### 문제점

1. **코드가 그리는 구분선** (`ctx.strokeStyle = "#222222"`, 2px): 템플릿의 구분선(y=769-770, ~5px 두께의 진한 검정)과 스타일이 다름

2. **섹션 사이 구분선 없음**: 템플릿 원본에는 각 섹션 사이에 얇은 회색 선이 있지만, 코드의 동적 영역에서는 섹션 간 구분선을 그리지 않음

3. **하단 템플릿 영역과의 연결부**: 동적 컨텐츠(흰 배경)가 끝나고 하단 템플릿(전형절차 + 복리후생)이 시작되는 지점에서 스타일 차이가 발생

### 해결 방안

1. 각 섹션 사이에 **얇은 구분선** 추가 (템플릿 스타일에 맞춰)
2. 상단 구분선 두께/스타일을 템플릿과 일치시킴
3. 컨텐츠 영역에 좌우 **테두리(border)** 추가 (템플릿에 테두리가 있는 경우)
